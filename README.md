# Minimal Project Management Tool

A basic project management tool featuring user authentication, project creation, and task tracking.

## Features List

- **Authentication**: JWT-based register/login with hashed passwords via Custom User Model.
- **Projects Management**: Authenticated users can Create, Read, Update, and Delete projects.
- **Tasks Management**: Tasks belong to projects with full CRUD capabilities.
- **Filters & Search**: Tasks can be filtered by status. Projects can be searched by title and description.
- **Modern UI**: Fully responsive Dashboard and Project Tracking UI using React, TailwindCSS v4, React Hook Form, Yup, and Zustand.

## Technology Stack

- **Backend**: Python, Django, Django REST Framework, Simple JWT, SQLite 
- **Frontend**: React 19, TypeScript, Vite, Tailwind CSS 4, Zustand, React Router DOM, Axios, React Hook Form, Yup

## Setup Instructions

### Backend Setup

1. **Navigate to project directory**: 
   ```bash
   cd project_management_tool
   ```

2. **Create a Virtual Environment**:
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```

3. **Install Dependencies**:
   ```bash
   pip install django djangorestframework djangorestframework-simplejwt django-cors-headers django-filter
   ```

4. **Apply Migrations**:
   ```bash
   python manage.py migrate
   ```

5. **Start the Development Server**:
   ```bash
   python manage.py runserver
   ```
   Backend runs at http://localhost:8000.

### Seed Database

Run the provided seed script to populate the database with mock accounts, projects, and tasks:

```bash
python seed.py
```

*Default users generated by seed script:*
- `test1@example.com` / `password123`
- `test2@example.com` / `password123`

### Frontend Setup

1. **Navigate to Frontend Directory**: 
   ```bash
   cd project_management_tool/frontend
   ```

2. **Setup Node Version**:
   Make sure you have Node 18+ installed.

3. **Install Dependencies**:
   ```bash
   npm install
   ```

4. **Start the Development Server**:
   ```bash
   npm run dev
   ```
   Frontend runs at http://localhost:5173.

## Known Limitations
- Pagination is implemented on the API layer but the frontend dynamically handles lists locally for simplicity instead of cursor-based pagination.
- Forms don't feature advanced debouncing for live searches yet.
- Tasks search is available from API but UI currently displays a filter dropdown for statuses.
- Due dates in forms use basic date-pickers standard to the browser.
- Uses SQLite for simplicity, but can be easily switched to PostgreSQL in settings.py depending on production requirements.
